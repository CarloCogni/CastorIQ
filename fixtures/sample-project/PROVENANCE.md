# Sample-project asset provenance

Recommended sources and licensing for the sample IFC + documents bundled
with every new beta user. None of the binary files are checked in — they
must be downloaded from the listed sources and dropped into this folder
before deploy.

## IFC model — `building.ifc`

**Recommended:** BuildingSMART Sample Files / Duplex Apartment (IFC4).

- Source repository: <https://github.com/buildingSMART/Sample-Test-Files>
- Specific file: any of the `Duplex_*.ifc` variants (mid-complexity, ~4–10 MB)
- License: CC-BY 4.0 — attribution OK in the help modal
- Why it works: fully-MEP'd small residential, plenty of psets, Storey/Space
  hierarchy intact, fire/thermal/acoustic properties realistic — exercises
  every Castor write-back tier without being huge.

**Fallback:** NIST FCM-test or KSC ER-7 (single-storey, simpler).

After download:

```bash
mv ~/Downloads/Duplex_A_20110907.ifc fixtures/sample-project/building.ifc
sha256sum fixtures/sample-project/building.ifc >> fixtures/sample-project/PROVENANCE.md
```

Pin the hash here once chosen so a re-deploy can verify the fixture.

## Documents — `docs/*.pdf`

Three short PDFs (~1–5 pages each) covering different domains so the RAG
retrieval has something meaningful to cite. Suggested mix:

- `fire-strategy.pdf` — fire-rating table, escape distances, compartment
  definitions. Plenty of "shall" / "minimum" requirement language.
- `thermal-spec.pdf` — U-values, glazing performance, building-fabric
  targets.
- `acoustic-notes.pdf` — separating-wall ratings, floor/ceiling treatments.

Anything in the public domain (or your own non-confidential teaching
materials) works. The provisioning command does not parse them — it just
ingests them into the documents pipeline.

## Pre-indexed embeddings (optional, recommended for prod)

For the deployed VPS we pre-compute embeddings for the sample project on a
beefier machine and ship the pgvector rows as a Django data fixture. The
provisioning command checks for `fixtures/sample-project/embeddings.json`
and uses it if present, otherwise falls back to recomputing on the VPS.
This is cheap-to-build but expensive-to-skip on a CPU-only Ollama deploy.

The embedding fixture is generated by the dev machine once and committed
to the repo (it's small — ~1 MB for the sample), then never recomputed in
production. Lands in M4.2.
