How the Asset Register Works

What is an asset?

An asset is a facility-management record for a single physical item — manufacturer, serial, warranty, condition, responsible person, classifications, notes. It can represent an item that is in your IFC model (a linked asset) or one that isn't in the model (an orphan — e.g. fire extinguishers, loose furniture, vehicles, post-handover replacements, sub-components like an AHU filter).

Three ways to add assets

Add assets Promote IFC entities. Opens a drawer listing every IFC entity in the project that doesn't yet have an asset record. Pick the ones you want to track and click Promote selected. No data entry required — the asset inherits name, type and location from the IFC entity.
Add manual asset Create an orphan. For items not in the IFC model. Only name is required. You may optionally add a free-text type/category (for grouping later), pin the item to a room/floor in the spatial tree, or give it a free-text location. Use this for consumables, mobile equipment, late-arriving replacements.
Import CSV Bulk load. Each row is either linked (has a global_id matching an IFC entity) or orphan (has name; everything else is optional). Runs dry by default — you review counts and errors before committing.

Linked vs orphan

Concern Linked asset Orphan asset
Identity IFC GlobalId Internal UUID (+ optional asset tag)
Display name IFC entity name Name field (required)
Type / category Read from IFC entity type Optional free text (grouping only — any label works)
Location Via the IFC entity's spatial container Optional spatial container (room/floor) + free-text fallback
Show in 3D Yes No — shows a Not in IFC model badge

CSV columns

Column Required Notes
global_id Linked rows only Must match an IFC entity in this project. Row is rejected if no match.
name Orphan rows only Display name for the orphan.
ifc_type No Optional free-text category for grouping — e.g. Fire extinguisher, IfcUnitaryEquipment, Service vehicle.
spatial_global_id No Orphan-only. GlobalId of an IfcSpatialElement (room/floor) in this project.
location No Orphan-only free-text location fallback.
asset_tag No Human-readable code (e.g. AHU-04). Unique per project when set.
manufacturer, model_number, serial_number No Free text.
commissioning_date, warranty_end No ISO date (YYYY-MM-DD).
classification_system, classification_code No (pair) Both required together. Missing (system, code) pairs are created on the fly.

Import runs dry-run by default — rows are validated inside a savepoint and rolled back so you can review counts and errors before committing.

Caveats

An IFC entity can have at most one linked asset (uniqueness constraint). Orphans can't be viewed in 3D until they're manually linked to an IFC entity (link action coming later). This form does not modify geometry or create IFC entities — it attaches FM metadata only. If an item ends up in the IFC model later, convert the orphan rather than creating a second record (no duplicates).
