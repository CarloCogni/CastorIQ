How the EVM Dashboard Works

What is Earned Value Management?

EVM is the industry-standard method for measuring project performance. It compares what was planned (Planned Value), what was actually earned (Earned Value), and what was spent (Actual Cost) at any point in time. The ratios — SPI and CPI — give an objective answer to "are we on track?".

Key Performance Indicators

KPI Formula Meaning Green / Red
SPIEV ÷ PVSchedule efficiency — how much work earned vs planned≥ 1.0 / < 1.0
CPIEV ÷ ACCost efficiency — earned vs actual spend≥ 1.0 / < 1.0
EACBAC ÷ CPIEstimate at Completion — projected final cost≤ BAC / > BAC
VACBAC − EACVariance at Completion — over/under budget forecast≥ 0 / < 0
TCPI(BAC−EV) ÷ (BAC−AC)Efficiency needed for remaining work to hit budget≤ 1.0 / > 1.10

Dashboard Sections

S-Curve

Weekly cumulative PV (planned, blue dashed), EV (earned, green), and — when actual cost is imported — AC (actual cost, amber). The gap between EV and PV is the schedule variance; between EV and AC is cost variance. Hover for exact values at any week.

WBS Performance

Per-stage SPI coloured green/amber/red. Shows which construction packages are ahead, on track, or behind plan. Based on task count weighting, not cost.

Trend Analysis

Weekly SPI and CPI sparklines show whether performance is improving, stable, or declining over the last 12 weeks. Linear regression projects the trend 4/8/12 weeks ahead. TCPI flags whether budget recovery is realistic. The Schedule Recovery Index (TSPI) shows how much SPI must improve, and whether that improvement is achievable at the current trend rate.

Delay Distribution

Tasks bucketed by overrun severity — from On Time to 31+ days late. Hover a bar to see example tasks in that bucket.

Forecast Completion

SPI-adjusted finish date shown on a timeline strip alongside the baseline. Variance is positive when the project is expected to finish early.

Project Controls Chat

Ask free-form questions — "Which packages are at risk?", "When will we finish?", "Which deadlines are in danger?". The LLM has access to all live EVM metrics, trend analysis, critical path, WBS risk scores, and constraint data.

Schedule Audit — Section Mismatches

Quality gate that compares each activity name against its planner-coded CSI MasterFormat division and flags likely mis-codes for human review. Two stages: (1) fast keyword pre-filter narrows ~10 k tasks to candidates in milliseconds; (2) LLM classifies each unique candidate name and assigns a verdict — Confirmed mismatch, Uncertain, or Likely OK (keyword false-positive). The planner's original CSI code is never changed; all flags are advisory. Correct mis-coded activities in P6 and re-import to resolve.

Cost Basis

Performance mode: when schedule costs or QTO estimates exist, Castor runs monetary EVM (cost_evm). When neither is available, it falls back to schedule performance mode — BAC is weighted by task durations and SPI reflects progress vs plan, not dollars. The mode badge above the KPI cards shows which mode is active.

Planned cost comes from P6 resource assignments; uncosted tasks are supplemented by QTO quantity estimates when available, then by task durations as a last-resort proxy (a banner warns you when a proxy is active — these are not monetary EVM numbers).

SPI forecast is computed server-side with the same guards as the Decision Summary: suppressed when no earned value is recorded, when SPI is non-positive, or when the extrapolated date exceeds a sane horizon (3× planned duration past baseline).

Actual Cost (AC) is read directly from P6ResourceAssignment.actual_cost — real imported data only. When actual cost has not been imported, AC is absent and CPI, CV, EAC, and VAC are disabled with a clear notice on each KPI card; they are never estimated from a formula. Import resource assignments that include actual cost to activate these metrics.

Caveats

SPI trend convergence: point SPI tends toward 1.0 near project end regardless of performance — treat trend forecasts cautiously beyond 75% completion. CPI, CV, EAC, and VAC are disabled entirely — not estimated — when P6 actual cost has not been imported. The KPI cards show a notice rather than a placeholder value. TCPI requires at least some AC < BAC; if AC has exceeded BAC the denominator is zero and TCPI is undefined. Constraint / deadline data (TCPI, MFO flags) requires a P6 XML re-import after the constraint fields were added — older imports have no constraint data. Trend regression uses a 12-week window. Fewer than 3 weeks of history will show "insufficient data" — this resolves automatically as the project progresses.
