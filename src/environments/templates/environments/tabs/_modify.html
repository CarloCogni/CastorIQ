Tier 1 – GREEN Single property or attribute changes. Auto-detected.

Click any example to paste it into the input. Castor will preview the changes before writing anything.

Change a property value SET_PROPERTY
Add a new property ADD_PROPERTY

Standard property sets (Pset_WallCommon, Pset_DoorCommon, etc.) are auto-created if they don't exist yet.

Remove a property REMOVE_PROPERTY
Rename or change attributes SET_ATTRIBUTE

Name, Description, ObjectType, and Tag are direct IFC attributes — not properties inside a Pset.

Multiple changes at once CHAIN

Use "and" to combine simple changes. Each becomes a separate proposal you can approve together.

Tips for best results
  • Be specific about entity types — say "wall" not "element"
  • Use exact property names when you know them (e.g. "FireRating" not "fire rating")
  • Specify the floor/level to narrow scope: "...on Level 1"
  • Use entity names for single-entity changes: "wall W-01"
  • One change at a time works best — or use "and" for simple combos
Tier 2 – ORANGE Multi-step plans. Auto-detected when the request needs more than simple property changes.

These requests require multiple coordinated operations. Castor generates a step-by-step plan that you review before anything is written. Click any example to try it.

Create a property set ADD_PSET

Creates new property sets with initial values. Use for tracking, compliance, and custom metadata.

Remove a property set REMOVE_PSET
Classify elements SET_CLASSIFICATION
Assign materials SET_MATERIAL
Compound operations PLAN

Castor decomposes complex requests into ordered steps. You review the full plan before approving.

Tips for Tier 2
  • Describe the goal, not individual steps — Castor will decompose for you
  • Mention the classification system name for classification ops
  • Material names should match your project conventions (e.g. "Concrete C30/37")
  • You can mix property changes with structural ops in one request
  • Review each step in the plan — you approve the whole plan at once
