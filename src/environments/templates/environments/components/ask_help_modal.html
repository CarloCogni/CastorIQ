How Ask works

What is it?

A natural-language interface to the parsed IFC model and any documents you've uploaded. Ask narrows the question down to relevant entities and document chunks via vector search, packs them into the LLM's context window, and answers with citations. Nothing here mutates the model — that's Modify.

How to ask a good question

  • Be specific. "What's the fire rating of the doors on Storey 2?" beats "tell me about doors".
  • Reference what you mean. Mentioning a property name, an IFC class, or an entity name helps the retriever find the right chunks.
  • Compare against docs. "Does the IFC's wall U-value match the thermal spec?" works because both sides are in the index.
  • Watch the citations. Every answer cites the IFC entity or document page it came from. If something looks off, click through.

Provider + token budget

Ask routes through whichever provider the operator has configured for this pipeline (Ollama / Anthropic Claude / Groq). Cloud providers count against your daily token cap shown at the top of this page; Ollama is free. The cap resets at midnight UTC — email the operator to request a raise.

Caveats

  • If a question doesn't match anything in the index, Ask says "I don't know" rather than guessing. That's deliberate.
  • Long conversations consume context. If the indicator at the top says you're approaching the limit, use the Compact button to summarise old turns.
  • Ask doesn't run scripts and can't change anything. For modifications, switch to the Modify tab.
