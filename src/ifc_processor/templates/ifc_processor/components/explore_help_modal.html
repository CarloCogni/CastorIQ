How Explore Works

What is Explore?

A read-only inspector for the parsed IFC model. You can walk the spatial tree (project → site → building → storey → space), filter the entity table by IFC class, search by name / GlobalID / tag / description, and open any entity to see its full attribute set, property sets (Pset_*), quantities (Qto_*) and type-level properties. Nothing here mutates the IFC file — that lives in Modify.

Layout

Spatial tree Left column. The IFC spatial decomposition (IfcSite → IfcBuilding → IfcBuildingStorey → IfcSpace). Click any node to filter the table to entities contained below that node.
Entity table Right pane. One row per occurrence (IfcElement) within the current spatial scope, paginated 50 at a time. Columns: Type · Name · Tag · Description · Element Type · Container · GlobalID.
Detail panel Modal on row click. Header (type, name, GlobalID), IFC Description + Tag, AI Summary, and every stored property set, quantity set, and type set.

How to use it

Filter by type Click an IFC type chip above the table (e.g. IfcDoor, IfcWall). Counts reflect the current spatial scope. All clears the filter. Use Schedule this type to jump to the type-grouped schedule for the active class.
Search The search box matches substring across name, GlobalID, tag and IFC description. Results respect the active spatial scope and type filter.
Keyboard ↑ / ↓ moves between rows; Enter opens the detail panel. Disabled while the search box has focus.
Ask / Modify Inside the detail panel, Ask about this jumps to the Ask tab with a pre-filled question; Modify jumps to Modify with the entity referenced. Both close the inspector first.
Export CSV The CSV button exports exactly the current filter — spatial scope, type chip, and search query all apply. Columns: GlobalID, Type, Name, Description, Tag, Element Type, Spatial Container.

Description vs AI Summary vs Tag

Field Source Use
Description IFC attribute IfcRoot.Description, parsed verbatim. Authoring-tool free text. Often empty.
Tag IFC attribute IfcElement.Tag, parsed verbatim. Source-system identifier (e.g. Revit element id). Not the FM asset tag.
AI Summary Generated by the embedding pipeline. Semantic blurb used for vector search in Ask mode. Shown in the detail panel under a blue AI Summary heading.

Caveats

Read-only. To change anything in the model, switch to Modify. Geometry isn't rendered here — Explore shows attributes and properties, not the 3D view. Only entities that survived parse appear. Rejected rows live in the admin IFCDataIssue log. Search is substring, case-insensitive, and joined as OR across name / GlobalID / tag / description. No regex. If the file selector shows nothing, no IFC file in this project has finished parsing yet.
