How usage and limits work

What are these two strips?

The two banners at the top of this page summarise your account's two consumption surfaces during the beta: storage (files you've uploaded plus the per-project Git history of your Modify approvals) and tokens (LLM input + output for the day, only for calls that draw from the host's shared pool).

Storage

Files — IFC models and documents you've uploaded across all your projects. History — per-project Git repositories that track every IFC modification you approve. Grows as you use Modify. Default quota: 1 GB. Recalculated when you visit this page. Email the operator to request an increase.

Tokens — the 50,000/day cap explained

What counts: input + output tokens for every LLM call routed through the host's shared pool ($150 of pre-paid Anthropic + Groq credit funding the beta). What doesn't count: calls using your own Anthropic / Groq API key (BYOK), and any call routed through local Ollama. Reset: daily at midnight UTC. Per-call visibility: the Ask and Modify chat input bars show a "Sending to: …" badge so you always know which path the next call will take.

The banner only renders when at least one of your purposes (Ask or Modify) still routes through the shared pool. If you've set both to BYOK or to Ollama, the cap doesn't constrain you and the banner disappears — that's intentional, not a bug.

Three ways calls get routed

Shared Managed (host pool). The default for new accounts. Counts toward your 50K/day cap. Anthropic Claude for Ask, Groq Llama for Modify.
BYOK Your key. Paste your own Anthropic or Groq key in Settings → Bring Your Own Key. Bypasses the cap entirely — calls bill directly to your provider account.
Local Local Ollama. Inference runs on the host's own Ollama instance. Free, no cap, no provider. The operator can flip individual purposes to Ollama from the admin in a single click.

Caveats

Storage is recalculated when this page loads; elsewhere it shows the last cached total — small lag after a fresh upload is normal. The token cap is a soft second layer: the hard limit is whatever's left of the host's pre-paid provider balance. If the balance runs out, every managed call gets a friendly "paused for maintenance" message until the operator tops up. BYOK auth failures (wrong key, rate limit) surface as a chat message pointing back to Settings — they never silently fall back to the shared pool.
