How Settings Work

What is this page?

Settings is where you manage your account preferences and review how this Castor build is configured. During the beta, most engine-level options (LLM provider, model, embedding model, daily token cap) are managed by the host — this page only exposes what's actually yours to change.

What you can change today

Theme Dark or light. Saved per user; flips live on toggle and persists across sessions via cookie.
Account Read-only. Username, email, and join date. Email and username changes go through the host during the beta — write to the address on the release notes.

Host-controlled in the beta

LLM provider and model. Anthropic Claude for Ask, Groq Llama for Modify — set centrally, not per user. Embedding model. Locked to mxbai-embed-large; changing it would require re-embedding every project. Daily token cap. Shared budget shown in the banner above the Ask and Modify chat inputs. RAG tuning (chunk size, top-K, similarity threshold) — fixed defaults during the beta.

What's coming

Bring Your Own Key (v1.1). Plug in your own Anthropic or Groq API key to bypass the shared cap. Encrypted at rest, scoped to your account. RAG tuning. Per-user chunk size, top-K, and similarity threshold. Notifications. Opt-in email for processing errors and write-back conflicts.

Caveats

The beta runs on shared infrastructure. Don't upload data you wouldn't want a host operator to see in error logs. Token usage is tracked at the chat banner, not on this page — you can see remaining budget where you're spending it. Sections marked Coming soon are visible on purpose — they signal what's on the v1.1 roadmap, not features that are silently broken.
