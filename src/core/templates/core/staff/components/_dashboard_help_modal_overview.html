How the Overview tab works

What is it?

The operator-facing view of Castor's runtime — LLM cost, token spend, error rate, latency, and feature engagement. It exists so the operator can spot abuse, runaway cost or a broken provider in under thirty seconds during the beta. All numbers are read live from existing tables (LLMCallLog, UserTokenBudget, ErrorLog) — no caching.

The six KPI tiles

Tile What it counts
Calls Total LLMCallLog rows in the selected window (24h / 7d / 30d).
Tokens Sum of tokens_in + tokens_out across every logged call.
Cost USD Sum of estimated_cost_usd — provider price table × token counts at write time. Estimate; reconcile against the provider dashboard before invoicing anyone.
Active users Distinct user_id in the call log over the window.
Errors Failed LLM calls (succeeded=False) plus any ErrorLog entries in the window. Yellow when non-zero.
p95 latency 95th-percentile latency_ms across all logged calls in the window. Returns — when no calls have a measured latency yet.

The two charts

Both charts span 30 days regardless of the KPI window — short windows are too sparse to see provider trends.

Tokens / day One stacked area per provider (Ollama / Anthropic / Groq). Spikes that appear in a non-Ollama series are paid traffic — that's where margin lives or dies.
Cost / day Same shape, USD on the y-axis, with a dashed red line at the daily-budget threshold (currently $10/day — adjust in OverviewView.daily_cost_threshold_usd). Crossing the line is the trigger to investigate that day's top users.

Top users by cost

Top 10 by USD cost in the selected window. BLOCKED badge means UserTokenBudget.hard_blocked = True — that user is currently refusing all cloud calls. Click the arrow icon to jump straight into Django admin for that user (groups, last login, password reset).

System pulse strip

The thin grey strip at the top is a cheap-counts summary — calls last hour (any LLM activity ticking over), unresolved errors (ErrorLog.is_resolved = False) and hard-blocked users. The healthz button next to the refresh icon opens /healthz/ as JSON in a new tab — that's the actual db + Ollama liveness probe.

Caveats

Costs are estimates — reconcile against provider dashboards monthly before any external reporting. Active users counts only people who fired at least one LLM call. Logged-in users who only browsed don't appear. The 24h window is rolling — last 24 hours from now, not "today". The page is staff-only and never linked from the public site. The navbar entry is gated by {{ '{% if user.is_staff %}' }}. p95 falls back to "—" when fewer than two calls have a recorded latency. Don't over-interpret single-day spikes.
