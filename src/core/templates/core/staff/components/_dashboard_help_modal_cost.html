How the Cost & Usage tab works

What is it?

Where the money goes, beyond the raw totals on Overview. Same data source (LLMCallLog) — different lenses. The trend charts span 30 days regardless of the window selector; short windows hide the patterns this tab is built to surface.

The four sections

Section Read
Cost by purpose (Ask vs Modify) Splits the cost stack into RAG (Ask) vs write-back (Modify). Useful when total cost rises — tells you which mode drove it.
Tokens local vs paid Ollama tokens collapsed under local; everything else (Anthropic, Groq, future cloud providers) under paid. Any drift of traffic from green toward red silently erodes Castor's gross margin.
Cost per active user, daily Total daily cost ÷ distinct active users that day. Vanity-resistant — a flat line as the user base grows means per-user spend is stable. A rising line means each user is costing more; that's the trigger to look at the budget heat strip below.
Per-user budget heat strip One bar per recently active user (last 7 days, or non-zero used_today right now). Bar colour: green <50%, yellow 50–80%, red ≥80% or hard_blocked. BLOCKED badge means cloud calls are being refused for that user.

"Active" means

The KPI strip's active users tile counts distinct user IDs that fired at least one LLM call in the selected window. The budget strip uses the same definition with a fixed 7-day lookback OR a non-zero used_today — so a user who's at 60% of their cap right now still appears even if they haven't fired a call in the last week.

Reading the budget bars

< 50% — comfortable headroom.
50–80% — keep an eye on it.
≥ 80% — close to the daily cap; user may be cut off mid-session.
BLOCKED hard_blocked = True — cloud calls already refused regardless of cap.

Caveats

Costs are estimated — provider price table × token counts at write time. Reconcile against provider dashboards monthly. used_today is reset lazily on the first call after midnight UTC. A user who hasn't fired a call yet today may still show yesterday's residual until they do. Unlimited cap (daily_cap = 0) renders as ∞ with no bar — the user can never hit a percentage. The trend charts always span 30 days. Use the window selector for the KPI strip, not the trend lines.
