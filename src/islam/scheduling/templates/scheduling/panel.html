How TimeLiner works

What is it? TimeLiner links your construction schedule to IFC model elements — exactly like Navisworks TimeLiner — enabling 4D simulation of your build sequence.

Workflow:

  1. Data Sources — upload a schedule file (.xlsx, .xer, or .xml). The system parses it and uses AI to validate it looks like a real construction programme.
  2. 4D Link — the unified linking, simulation, and review workspace. Run Castor Link Engine (5-layer: exact → normalised → heuristic → embedding, with LLM schedule context analysis) or match by IFC property name. The 3D viewer shows the model in real time; drag the 4D timeline slider to animate the build sequence. Review Links loads every binding — accept, remove, bulk-accept at ≥ 95% confidence, or manually add links. Export the binding table as CSV from the Review panel.
  3. EVM — Earned Value Management dashboard. Shows SPI, CPI, EAC, and VAC KPI cards plus a cumulative S-curve (PV, EV, AC). Uses task costs when available; falls back to duration weighting when no costs are in the schedule.
  4. Look-ahead — Rolling window showing what starts, is in progress, and finishes across the next 2–6 weeks. Critical path tasks are flagged with a CP badge. Click a week card to expand/collapse it.
  5. Intelligence — AI-powered Q&A over your schedule. Ask natural-language questions such as "Which tasks are delayed?" or "Summarise MEP progress." The system embeds all tasks into a vector store and retrieves the most relevant ones as context for the LLM. Click Embed Tasks before asking for the first time, or after uploading a new schedule.

Caveats: Castor Link Engine matching is probabilistic — always review heuristic and embedding results via Review Links before relying on 4D simulation. Elements must be processed (IFC Insights complete) for best coverage. EVM actual cost (AC) comes directly from P6 resource assignments; CPI, EAC, and VAC are disabled (shown as N/A) when no actual cost has been imported — they are never estimated or synthesised. Intelligence answers are only as good as the schedule data — always verify LLM output against your programme.
