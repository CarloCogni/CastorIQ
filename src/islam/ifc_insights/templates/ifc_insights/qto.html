How Quantities works

What is it?

The Quantities tab performs a Quantity Take-Off (QTO) against your IFC model — extracting areas, volumes, and lengths from standard Qto_*BaseQuantities property sets. It aggregates quantities by element type, level, and material, and supports optional unit costs for 5D cost estimation.

Workflow

  1. Recompute — scans the latest processed IFC file and populates the QTO cache. Run this after uploading a new IFC or when quantities look stale.
  2. Review Coverage — elements with Qto_*BaseQuantities pset data are marked ifc (reliable). Elements without any Qto pset are counted only (estimated). The coverage % shows what fraction has real IFC quantities.
  3. Set Unit Costs — type a cost per unit in the Unit Cost column (e.g. currency per m² or m³). After saving, click Recompute to apply the costs and unlock the Total Cost column and EVM integration.
  4. Export — download full QTO data to Excel: Summary by Type, By Level, and per-entity Detail sheets.

Reading the table

  • Amber rows — element types where fewer than 50% of elements have IFC quantity data. Totals for these types are incomplete.
  • Click a row — expands to show the top 5 individual elements for that type.
  • Click a column header — sorts the table by that column. Click again to reverse order.
  • Unit Cost input — type a value and press Enter (or click away) to save. Cost totals are applied on the next Recompute.

Caveats

QTO extraction relies on exporters writing standard Qto_*BaseQuantities psets. Models exported without these psets will show low coverage. Unit costs are stored per IFC type (e.g. IfcWall, IfcSlab) and applied uniformly — not per-element. After editing unit costs, click Recompute to refresh cost totals and propagate changes to the EVM tab. The EVM tab uses QTO-derived costs to fill in task costs when the schedule has no cost data attached.
