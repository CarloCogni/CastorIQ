How Levels Work

What is the Levels Panel?

The Levels Panel is the authoritative registry of floor levels for your building model. It reads IfcBuildingStorey objects from your processed IFC file, shows reference coordinate points, flags storeys missing from your schedule, lets you add manual levels, and can write everything back into the IFC file.

Panel sections

Reference Points — Reads IfcSite RefElevation and named entities ("Project Base Point", "Survey Point") from your IFC model so you know the coordinate datum.
IFC Storey Registry — Every IfcBuildingStorey in the model, sourced from the database. Shows element count and whether the storey name appears in your schedule. Use Import to add a storey to the Level Registry so you can edit its name and Z-elevation and write it back via Apply.
Missing Level Hints — Appears when your schedule tasks reference level keywords (e.g. "Level 3", "GF", "Basement") that do not match any storey name. Import or create matching levels to close the gap.
Manual Levels — Add levels that have no counterpart in the IFC file. Apply will create new IfcBuildingStorey objects for these.
Apply to IFC — Writes all linked storeys (name + Z) and creates new storeys for manual levels. Shows a diff preview before you click. A .ifc.bak backup is created automatically and a Git commit is recorded in the project audit log.

Typical workflow

  1. Process your IFC file (IFC Insights tab) so the storey registry populates.
  2. Click Import next to each storey you want in the Level Registry.
  3. Edit any storey name or Z-elevation inline with the pencil icon.
  4. Check the Missing Level Hints alert and add manual levels to cover any gaps.
  5. Click Apply to IFC to write changes back to the file.

Who can do what

  • Any project member can view the storey registry and reference points.
  • Import, edit, add, and Apply all modify project data — treat Apply like a write-back operation.

Caveats

Apply overwrites the IFC file in place. A .ifc.bak sibling is created first but is not itself versioned. Only storey name and Z-elevation are written back — slab geometry and thickness are out of scope. The In Schedule badge is based on a simple text-match against task names and activity codes — it is a hint, not a guarantee. Unlinking a storey from the registry removes the IslamLevel record but does not modify the IFC file.
