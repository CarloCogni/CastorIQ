How IFC Issues works

What is the IFC Issues tab?

IFC Issues gives you a targeted audit of data quality problems in your model — specifically which elements are missing the properties needed for 4D and 5D workflows, how your Activity IDs relate to the schedule, and whether your floor levels are correctly populated and matched to tasks. Each section loads independently so slow sections never block faster ones.

The four sections

Missing Activity ID — Every IFC element that has no Activity ID property (or has an empty value). These elements cannot be linked to schedule tasks. Use Copy Global ID to grab the identifier, or Open in 3D Viewer to locate the element in the model.
Missing Cost Data — Elements with no Cost or Unit Cost property. These are excluded from 5D cost totals.
Activity ID & Name Audit — All Activity ID / Activity Name pairs found in element properties, grouped and counted. The Linked to Task? column shows whether any element in that group has a TaskEntityBinding in the schedule: ✓ = linked, ⚠ = Activity ID exists but no binding found.
Levels Health — Every IfcBuildingStorey with its element count, schedule match, and a status summary: ✓ OK = has elements and matched in schedule, ⚠ Not in schedule = has elements but no task match, ✗ No elements = storey exists but contains no entities. Use Fix in Levels → to jump to the Levels panel.

Typical workflow

  1. Check Missing Activity ID — export the CSV and share with your BIM team to populate the property in their authoring tool, then re-upload and re-process the IFC.
  2. Check Activity ID & Name Audit — confirm every Activity ID group is linked to a schedule task. Unlinked groups (⚠) should be re-run through the TimeLiner auto-link.
  3. Check Levels Health — storeys with no elements indicate spatial containment gaps; storeys not in schedule need task names that reference the storey name.
  4. The red badge on the tab heading shows the combined count of missing Activity ID + missing Cost elements. Aim for zero.

Who can do what

  • Any project member can view all four sections and export CSVs.
  • Fixing issues requires re-authoring the IFC in a BIM tool, re-uploading, and re-processing — typically done by the BIM manager.

Caveats

Sections load in parallel on page open — large models may take a few seconds per section. Activity ID matching uses a suffix check (.endswith("activity id")) so custom Pset prefixes like Castor_ActivityID are correctly detected. Missing Cost counts all elements without a numeric Cost or Unit Cost — elements with a Cost = 0 are treated as missing. The In Schedule check for Levels Health is a text-substring match — it is a hint, not a guarantee of a formal schedule link. Display is capped at 500 rows per section; use Export CSV for the full list.
