### [2026-05-14] islam/scheduling/services/autolink.py — Layer 3 spatial refinement
- Action: Modified
- What it does: Layer 3 heuristic now returns ALL entities of the matching IFC type (up to _MAX_HEURISTIC=50) instead of one best-named entity. If the task name contains a level/floor hint (e.g. "Level 3", "Ground Floor", "B1", "L01"), candidates are first narrowed to entities whose spatial_container storey name contains that hint; falls back to the unfiltered type list when no spatial match is found. Added _MAX_HEURISTIC constant, _extract_level_hint helper, and _build_container_names helper. spatial_container field added to entity_list .only() call.
- Why it was necessary: Returning a single best-named entity was too narrow for heuristic confidence (0.70); bulk-returning all same-type same-floor entities is more useful for the Phase 3 review UI and gives the reviewer a meaningful cohort to accept or reject.
- Depends on: IFCSpatialElement.entity (OneToOne), IFCSpatialElement.SpatialType.BUILDING_STOREY

### [2026-05-14] islam/scheduling/services/autolink.py
- Action: Created
- What it does: 4-layer auto-link pipeline. Layer 1: exact property match (confidence 1.0, auto-accept). Layer 2: normalized property match (0.95, auto-accept). Layer 3: IFC-type keyword heuristic (0.70, needs_review=True). Layer 4: embedding cosine similarity (≥0.82 auto-accept, 0.65–0.81 needs_review=True, <0.65 skip). Clears stale TaskEntityBinding rows on each run. Writes M2M ifc_entities only for auto-accepted bindings.
- Why it was necessary: Phase 2 of Schedule Linking — replaces the single-method LLM/param approaches with a layered pipeline that maximises auto-accept rate while flagging uncertain matches for human review.
- Depends on: TaskEntityBinding, Task, IFCEntity, IFCFile, EmbeddingService, pgvector CosineDistance, linker._read_property
