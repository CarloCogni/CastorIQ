How IFC Insights works

Readiness Rings — 4D measures accepted TaskEntityBinding coverage (trusted schedule-to-model links). 5D measures schedule cost via binding or IFC cost properties. Green ≥ 80%, Amber 40–79%, Red < 40%. Counts beneath each ring show elements still missing that coverage.

Schedule Progress ring — shows what fraction of physical schedule tasks are complete (status = Complete, or actual end date set). Use the selector beneath the ring to switch the weighting: By Task Count (equal weight), By Cost (proportional to task cost), By Duration (proportional to planned days), or Custom Weight (uses the per-task weight field). The selected mode is saved per project. Click any construction stage chip to filter progress to that stage only (Substructure, Structure, Envelope, MEP, Finishes, External Works). Chips only appear for stages that have at least one task in the project. Stages are auto-detected from task name keywords when tasks are imported.

View Issues → — opens the IFC Issues tab: IFC property audits (Activity ID, Cost), Activity ID & Name Audit, and Levels Health. Property gaps exclude elements with accepted Castor bindings.

Breakdown Cards — element counts grouped by Level, Element Type, and Material. Loaded live from the database.

Non-Physical Tasks — appears when the schedule contains tasks flagged as non-physical (procurement, admin, milestones, commissioning). Shows total count, how many were auto-detected vs. manually locked, top activity types, and stage distribution. Non-physical tasks are excluded from all readiness and progress calculations. Click “Review in schedule →” to open the schedule tab filtered to these tasks.

IFC Health Checks — detailed quality checks (severity, global ID, type, issue, suggested fix) are now in the IFC Issues tab under Section 0.

QTO badge on 5D ring — the blue QTO: X% coverage pill below the 5D Readiness ring shows what fraction of model elements have IFC quantity data computed. Click it to open the Quantities tab. If no QTO has been run yet, the badge shows “not computed”.

Engine Selector — Castor Engine uses the main analysis pipeline. Legacy Engine is experimental.
