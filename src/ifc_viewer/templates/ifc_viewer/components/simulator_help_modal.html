How Castor Simulator Works

What is Castor Simulator?

Castor Simulator is the 4D/5D control layer on top of the 3D viewer. It lets you colour-code elements by any dimension (material, level, type, schedule status), analyse schedule linkage gaps, animate the build sequence floor by floor, and scrub through a weekly construction timeline.

Color By Filters

None — restore default ifc-lite colours. Material / Level / Element Type — each unique group gets a distinct colour from a 12-colour palette. The legend appears in the viewer overlay. Schedule Status — green = entity has an Activity ID property linked to a schedule task; grey = not linked. Instant visual gap map.

Gap Analysis Panel

Opens a side panel grouped by Level, Element Type, or Material. Each row shows how many elements are linked to a schedule task versus total. Click any row to highlight the unlinked elements in the 3D view. Export the full table as CSV.

"Linked" means the entity has an Activity ID property — the same field the TimeLiner uses when auto-linking tasks. Import a schedule and run auto-link first for meaningful coverage numbers.

Build Sequence

Animates elements level by level in Z order (lowest floor first). Each floor fades in over 800 ms, pauses 1.2 s, then the next floor appears. Floors are ordered by their Z elevation from the Level Registry — add levels there for accurate ordering. When the animation ends all elements are shown in blue (complete).

Timeline Slider

Requires a linked schedule (import one in the TimeLiner tab first). The slider moves through weekly buckets. At each week: active tasks colour their linked elements green; completed tasks colour elements blue; unstarted elements are left at default.

Play auto-advances every 1.5 s (adjustable with the speed selector: 0.5× / 1× / 2×).

Caveats

Color By and Timeline both apply colours to the same model — the last action wins. Click Reset to restore defaults. Build Sequence uses the Level Registry Z values. If no levels are saved, floors sort alphabetically. Gap Analysis counts Activity ID coverage only — other schedule linkage methods are not reflected. Timeline performance depends on the number of tasks × linked entities per week bucket. Very large schedules may be slow to fetch.
