# WRITEBACK PIPELINE — AUTOMATED TEST PROMPTS
# ════════════════════════════════════════════════════════════════════════════
#
# Purpose ─────────────────────────────────────────────────────────────────────
#   Single, runnable spec covering every routing branch of the writeback
#   pipeline (Triage → Slots → Resolver → Tier Router) plus every reject
#   category the architecture documents. Designed so Claude Code can run it
#   end-to-end without manual UI typing and diff actual vs expected per stage.
#
# How to run ──────────────────────────────────────────────────────────────────
# <project loaded with Ifc4_SampleHouse.ifc>
#
#   cd src && uv run manage.py dry_run_v2_pipeline \
#       --project "5afb013a-4fa9-42a6-b78e-c9db4827bcca" \
#       --prompts-file ../docs/specs/writeback/pipeline-test-prompts.txt
#
#   The runner consumes one non-empty, non-`#` line as one prompt. Comments
#   above each prompt declare the expected output of each pipeline stage:
#
#     triage:    <segment count + kinds the TriageClassifier should emit>
#     slots:     <fields the SlotExtractor should fill, per segment>
#     resolver:  <mode + diagnostic the EntityNameResolver should produce>
#     router:    <final `→ tier N, operation X` OR `→ tier 0 REJECT: <reason>`>
#
#   For reject sections (13–17) the `router:` line declares a substring of
#   the rejection reason — exact wording can drift; the substring is what
#   we lock against.
#
# Test fixture (Ifc4_SampleHouse.ifc) ─────────────────────────────────────────
#   Three walls — Pset_WallCommon: Reference, IsExternal=true, LoadBearing=false,
#   ExtendToStructure=true, ThermalTransmittance=0.24. NO FireRating — exercises
#   SET_PROPERTY → ADD_PROPERTY auto-fallback.
#
#     Wall-A: IfcWall Basic Wall:Wall-Ext_102Bwk-75Ins-100LBlk-12P:285330
#     Wall-B: IfcWall Basic Wall:Wall-Ext_102Bwk-75Ins-100LBlk-12P:285395
#     Wall-C: IfcWall Basic Wall:Wall-Ext_102Bwk-75Ins-100LBlk-12P:285459
#
#   One window — Pset_WindowCommon: Reference, IsExternal=true,
#   ThermalTransmittance=5.56.
#
#     Window: Windows_Sgl_Plain:1810x1210mm:286105
#
# Section index ───────────────────────────────────────────────────────────────
#    1. Single-entity SET_PROPERTY                            (T1, scope=specific)
#    2. Multi-entity specific                                 (T1, specific_multi)
#    3. All-of-type                                           (T1, all_of_type)
#    4. Property-filtered                                     (T1, filtered)
#    5. SET_ATTRIBUTE                                         (T1)
#    6. ADD_PROPERTY / REMOVE_PROPERTY explicit               (T1)
#    7. Tier 2 PSET ops                                       (T2)
#    8. Tier 2 classification + material                      (T2)
#    9. Tier 2 compound plans (multi-segment PROPERTY/PSET)   (T2)
#   10. Tier 3 CREATE                                         (T3)
#   11. Tier 3 DELETE                                         (T3)
#   12. Tier 3 RELATIONSHIP / spatial reassignment            (T3)
#   13. T0 REJECT — geometry (HARD out-of-scope)              (T0)
#   14. T0 REJECT — non-modification (greetings, ask, empty)  (T0)
#   15. T0 REJECT — missing slots (no value / target / prop)  (T0)
#   16. T0 REJECT — entity not found / ambiguous              (T0)
#   17. T0 REJECT — pset inference fails                      (T0)
#   18. Drift / regression triggers (known prompts that previously broke) (mixed)
