02

Color tokens.

A two-layer system. Structural tokens (primary navy, secondary dusty navy, neutrals) handle chrome, surfaces, and most active states. Semantic tokens (burgundy, gold, ok-green) carry meaning — they're used sparingly and never for decoration.

The rule. Categorical color (e.g. tab identity) re-uses tokens from the structural and semantic palettes — we never invent new hues for category. Five named colors, intentionally repeated, beats nine bespoke ones for legibility.

05

Rules.

Five principles that hold the system together. When in doubt, default to these — they encode the decisions we kept circling back to.

01
Two color layers
Structural (navy family + neutrals) does the chrome, surfaces, and 80% of UI work. Semantic (burgundy, gold, ok-green) carries meaning and is used sparingly. Never mix the roles — a button never picks up burgundy unless it commits a destructive action.
02
Warm cream, cool accents, warm criticals
The whole identity. Backgrounds and text are warm; primary accents are cool blue; danger and warning stay warm. When you add a new component, ask: is this chrome (warm), action (cool), or alarm (warm)?
03
Categorical reuse, not invention
Tabs and section identity reuse the five named colors with intentional repetition. Ask + Explore both navy. History + Facilities both dusty navy. Never invent a new hue to distinguish a category — five legible signals beat nine confusing ones.
04
Neutral buttons
Categorical colors live on tabs, badges, and pills — not on buttons. Buttons are navy (primary), outlined panel (secondary), or ghost. A burgundy button means destructive action. A gold button doesn't exist.
05
Dark is derived, not redesigned
Dark mode keeps the warm-chrome character — it doesn't switch to cool black. Cream → warm charcoal, navy → lifted navy, criticals stay warm and brighten one step. Every dark token has a direct mapping in :root via body[data-theme="dark"].
06
Mono for codes only
JetBrains Mono is reserved for GUIDs, hex values, file paths, timestamps, and metadata eyebrows. Never use it for body copy or headings — the contrast with DM Sans is part of the signal: "this is a value, not a label."
06

Implementation.

CSS variables to paste into your stylesheet. Both themes wired through a single data-theme attribute on <body>.

/* === LIGHT THEME (default) === */
:root {
  --pri: #1B3A6B;   --pri-d: #122849;
  --sec: #5A7090;   --sec-d: #445872;
  --bur: #5A3D3D;   --gld: #A09880;
  --ok:  #3D8A5A;   --on-pri: #dde8f8;
  --ink: #1A1613;   --dk: #2A2520;
  --gr:  #6B6560;   --lt: #9A9AAE;
  --bd:  #E8E4DE;   --bg: #F8F6F2;
  --wh:  #FAFAF8;   --pl: #EFECE6;
  --f: 'DM Sans', system-ui, sans-serif;
  --mono: 'JetBrains Mono', monospace;
}
